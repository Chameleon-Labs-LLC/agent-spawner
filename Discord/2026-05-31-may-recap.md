🦎 **agent-spawner — what shipped in May 2026**

May was the **"Managed Agents API correctness"** cycle (PR #1) — we got the Managed Agents surface right, added a proper orchestrator pattern, and rewrote the docs to match how Managed Agents actually work. Released as **v0.1.1**. 👇

**✨ New: `--type orchestrator`**
The recommended Managed Agents SaaS shape. A managed agent + a client process that holds the SSE stream, dispatches tool calls to remote workers over the HMAC bridge, and feeds results back.
• 🔄 Lossless reconnect (event backfill + dedupe)
• 🧰 FastAPI worker template with HMAC-verified `/execute` (constant-time compare, 60s skew window, nonce LRU)
• 🪪 Optional `user_id` identity passthrough into session metadata

**📚 Docs rewrite**
• New **orchestrator-pattern** guide + four-type comparison table ("if you reached for *hybrid* and started inventing a verifier proxy, you wanted *orchestrator*")
• New **auth-and-identity** guide — how to thread real web users through to a managed agent, with a hard rule against putting secrets in prompts/messages/metadata
• **managed-vs-local** rewritten to describe Agents + Environments + Sessions accurately
• README now leads with the four agent types + full orchestrator setup

**🐛 Fixes**
• Corrected the API surface: real `/v1/agents` endpoint, `system` field, default `claude-opus-4-7`, dropped bogus `max_tokens`
• Fixed `events.stream()` await shape (it's an AsyncStream, not a context manager — was throwing `AttributeError` on first call)
• Pattern 5 idle gate now breaks only on `end_turn` / `retries_exhausted`, so it stops racing against post-tool-result idles
• 🔐 Scaffolder now splits committed `.env.example` (placeholders) from gitignored `.env` (real secrets) — closed a prior HMAC-secret leak

Net: the orchestrator type is the new front-door for Managed Agents, and freshly-scaffolded agents now work first-try. 🎉
